## 验证边界声明

- D0 阶段的全部命令执行、数据读取与配置修改验证，均限定在研究员自有租户 / 自有 Pod 范围内。
- 没有对其他用户 Pod 进行未授权的横向移动。
- 所有截图脱敏处理：Gateway Token、Session Token、JWT、容器 hostname、外部 IP、内部 ELB 域名、Solana 钱包公钥按需打码。
- D0 阶段研究在 2026 年 5 月独立完成，沿用研究员对 Donut 全产品线的负责任披露框架。

## 1. 控制面连接材料

**对应漏洞**：D0-1、D0-2、D0-13

### 截图 1：D0 环境接口返回控制面连接材料

> 脱敏后的环境接口响应，展示前端在普通登录态下可获得的字段集合，包括外部入口、Gateway 端口、Gateway Token 等连接控制面所需的关键材料。具体取值打码。

### 截图 2：控制面连接过程

> 脱敏后的 WebSocket 握手过程、认证阶段和控制面 UI 入口。展示从前端 Token → Gateway 接受连接 → 控制面 UI 加载的完整路径。

### 文字记录：验证结论

1. D0 前端环境接口返回的字段足以让前端直接连接到底层控制面 WebSocket。
2. 控制面 WebSocket 对来源 / Origin 的约束不足，连接后即获得超出普通产品界面的操作能力。
3. Gateway Token 的生命周期较长，缺少明确的强制轮换或单次使用语义。

### 影响声明

前端可见即可控制面可达，意味着"普通登录态 → 底层控制面"之间不存在足够强的隔离层。这条路径是 D0 全部后续问题（配置读取、文件写入、自有 Pod RCE、Session Token 提取）的入口。

## 2. OpenClaw 识别证据

**对应漏洞**：与开源依赖透明度有关的安全治理观察（详见 [README §安全治理观察](../README.md#security-governance)）

### 截图 3：OpenClaw logo

> 访问 D0 控制面后，页面明确显示 OpenClaw logo 的截图。

### 截图 4：OpenClaw 文档入口

> 控制面文档入口指向 OpenClaw 官方文档的截图。

### 文字记录：验证结论

1. D0 控制面来源不是基于"产品行为相似"的推断，而是通过 logo 和文档入口直接确认。
2. Donut 对外宣传 D0 时，没有把 OpenClaw 作为主要产品信息呈现。
3. OpenClaw 项目上线时间较短，但被作为 Donut 核心 AI Agent 产品 D0 的底层。

### 影响声明

本节不构成对 OpenClaw 项目本身的安全评价。相关讨论的目的是：在 D0 作为高融资 AI + Crypto 产品对外推广的背景下，应充分披露其对开源依赖的实际情况，并据此调整安全工程投入和漏洞响应机制。

## 3. 配置读取、修改与文件写入

**对应漏洞**：D0-4、D0-3、D0-5、D0-6、D0-7

### 截图 5：Agent / heartbeat / beforeRun 配置读取

> 脱敏后的控制面配置读取响应，展示可读取的字段范围（如 Agent 注册信息、heartbeat 调度、beforeRun 脚本入口、运行时配置等）。具体脚本内容打码。

### 截图 6：配置修改或文件写入能力

> 脱敏后的 `config.patch`、`agents.files.set` 或等价能力调用证据。展示请求接受范围、返回状态、变更可见性，但不展示真实写入内容。

### 文字记录：验证结论

1. 控制面在认证通过后开放了远高于"模型对话"所需的能力：读取、修改、写入。
2. `config.patch` 类接口缺乏明显的审计、变更保护或回滚记录。
3. `agents.files.set` 类接口允许向 Agent 工作区写入文件。
4. heartbeat / beforeRun 等运行时执行机制属于"配置一旦修改即被调度执行"的执行路径，本不应由前端可获得的普通会话材料触达。

### 影响声明

AI Agent 的安全边界不能只依赖模型层的"是否拒绝危险请求"。一旦控制面层暴露配置修改、文件写入和执行机制访问能力，无论模型本身多保守，攻击者都可以绕过模型层、直接通过运行环境实现执行。

## 4. 自有 Pod RCE 与环境信息读取

**对应漏洞**：D0-8、D0-9、D0-10、D0-11、D0-12、D0-14

### 截图 7：自有 Pod 内命令执行验证

> 在研究员自有 Pod 内执行的非破坏性命令的脱敏截图。命令范围限于：
>
> - 输出当前时间、用户、容器内核版本等系统标识；
> - 列出当前进程；
> - 查看挂载点和容器运行时；
> - 不涉及破坏性命令、不涉及网络扫描、不涉及对其他 Pod 的访问。

### 截图 8：Session Token / 环境变量 / 内部服务地址读取

> 在自有 Pod 内读取 Session Token、环境变量和内部服务地址的脱敏截图。所有真实 Token、JWT、ARN、ELB 域名和内部主机名打码。

### 文字记录：验证结论

1. 在自有 Pod 内取得命令执行能力后，能够稳定读取到 Session Token、容器环境变量和内部服务地址。
2. Session Token 文件由容器内进程可读，意味着 RCE 直接升级为"以用户身份访问后端 API"的条件。
3. 容器环境变量中暴露了生产标识、内部入口和运行时配置，构成纵深攻击面。
4. Pod 间网络隔离边界尚未被未授权探测，未来如经授权扩展，仍需评估横向风险。

### 影响声明

AI Agent 平台的执行环境一旦失守，影响远超传统"模型越狱"。它意味着：

- 攻击者可读取到用户在 Agent 环境中存放或经手的所有敏感资料。
- 攻击者可以"以用户身份"访问 Donut 后端 API，从而把 D0 阶段的执行环境问题接到 Donut Browser 阶段的资金风险攻击链上。
- AI 模型层的"危险请求拦截"在这条链路上完全失去作用。
