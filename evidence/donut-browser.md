## 验证边界声明

- 所有链上交易验证使用研究员自有 Solana 钱包完成，已完成 11 笔自有资金范围内的真实上链。
- 跨用户钱包归属校验测试中，请求由研究员自有账号发起、引用研究员事先构造的另一个研究员自有钱包，未涉及任何第三方真实用户资产。
- 与 Donut 后端的全部交互均在 Donut 官方 Bug Hunt 活动及后续沟通框架内完成。
- 本页所有截图均为脱敏后版本：钱包地址、交易哈希、会话 Cookie、Turnkey 凭据、内部主机名、内部 ARN 与 ELB 域名按需打码。

## 1. 1-click 静默交易链

**对应漏洞**：DB-1、DB-11

### 截图 1：自有钱包链上交易记录

> 自有钱包 / 自有资金范围内完成的 Solscan 交易截图或交易哈希列表。包括 JUPITER_SWAP、WRAP_SOL、TRANSFER、KAMINO_DEPOSIT 与 KAMINO_WITHDRAW 等不同类型交易，证明攻击链覆盖 swap / DeFi 存取等多种链上能力。

### 截图 2：交易请求与执行链路

> 脱敏后的请求路径、交易构建过程、后端执行结果以及最终落到链上的交易哈希。截图展示从一次访问触发开始，到 `action-mcp/query` 构建交易、`action-mcp/execution` 推进执行、`wallet/portfolio` 回填资产视图的完整后端链路，整个过程没有出现签名授权弹窗或二次确认。

### 文字记录：验证结论

1. 用户没有进行任何手动签名或授权确认。
2. 浏览器在一次访问 / 一次点击范围内携带会话向后端发起调用。
3. Donut 后端在没有先要求用户重新授权的情况下，自动完成交易构建并推进执行。
4. 交易最终成功上链，可在 Solscan 上看到记录。
5. 上述链路在多个交易类型（swap、wrap、transfer、deposit、withdraw）上均成立，已通过自有资金 11 笔交易验证。

### 影响声明

1-click 静默交易链意味着：攻击者只要让用户访问一次恶意页面或点击一次恶意链接（无需用户登录任何第三方网站、无需任何签名行为），就有条件转走该用户在 Donut Browser 中所有 Solana 资产。这是一个独立于第三方签名服务的、由 Donut 自身后端推进的资金风险。

## 2. 跨用户钱包交易构建与 AUTH001

**对应漏洞**：DB-1、DB-2

### 截图 3：AUTH001 拦截结果

> 使用研究员账号 A 构造来自研究员钱包 B（未绑定到账号 A）的交易请求后，Turnkey 在最终签名阶段返回 AUTH001 的脱敏截图。截图保留请求路径、AUTH001 错误码、原始响应体，但隐藏真实公钥、签名材料和 Turnkey 内部标识。

### 截图 4：钱包归属校验链路

> 脱敏后的请求字段、响应状态、Donut 自身 API 层和 Turnkey 策略层在调用栈中的相对位置。

### 文字记录：验证结论

1. Donut 后端没有在 API 入口阶段先校验"`from_wallet` 是否属于当前会话用户"。
2. 后端依然为非当前账号所属的钱包构建并提交了交易请求。
3. 请求最终送达 Turnkey 策略判断阶段后，被 AUTH001 拦截。
4. 真正阻断资金转出的是 Turnkey，而不是 Donut 自身的权限模型。

### 影响声明

资金安全边界依赖第三方签名服务在最后一刻的兜底，是一种把核心权限校验外包给签名供应商的设计选择。一旦 Turnkey 策略放宽、被替换或在某一交易类型上策略不覆盖，Donut Browser 的钱包归属校验缺失将立刻退化为可以跨用户构造资金转出的实质漏洞。

## 3. 资产与交易历史 IDOR

**对应漏洞**：DB-3、DB-23、DB-26

### 截图 5：非当前用户钱包资产读取

> 通过钱包、资产、持仓或交易历史相关接口，直接读取非当前账号所属钱包的资产组合、持仓成本和历史交易。截图为脱敏后版本，仅展示接口名、响应字段结构和数量级，钱包公钥和持仓明细打码。

### 截图 6：高价值目标画像风险

> 通过批量枚举发现的高价值钱包样本数量、资产总量分布等统计画像。截图不展示具体钱包公钥，仅展示能够说明"批量枚举存在且可识别高价值目标"的脱敏数据。

### 文字记录：验证结论

1. 钱包详情、资产、持仓和交易历史接口允许在仅持有研究员会话的前提下读取非当前账号所属的钱包数据。
2. 接口对枚举行为没有充分的速率限制和访问审计。
3. 读取结果可用于筛选高价值钱包和活跃交易策略。

### 影响声明

即使不直接转移资产，资产画像泄露也会显著提升后续钓鱼、定向攻击和交易操纵的成功率，并降低攻击者识别高价值目标的成本。它本身就是 Donut Browser 用户隐私和账户安全的实质损害。

## 4. CORS、MCP 与组合攻击面

**对应漏洞**：DB-4、DB-5、DB-20、DB-31

### 截图 7：CORS credentials 风险

> 脱敏后的 CORS 响应头截图，展示子域通配符 `Access-Control-Allow-Origin` 与 `Access-Control-Allow-Credentials: true` 组合出现在敏感接口的响应中，以及该组合在浏览器侧被触发携带登录态时的实际行为。

### 截图 8：MCP 工具边界问题

> 脱敏后的 MCP 工具枚举结果、工具调用入口和参数 schema。截图展示工具数量、命名空间、能力范畴，不展示完整的内部 prompt、token 或工具调用真实参数。

### 文字记录：验证结论

1. 子域通配符 CORS + `credentials: true` 在敏感接口上同时生效，扩大了浏览器侧信任边界。
2. MCP 工具层可被枚举到完整工具生态，工具参数 schema 公开，可用于构造攻击链。
3. 与 DB-6 ~ DB-10 等行为层漏洞组合后，远程触发成本显著下降。

### 影响声明

CORS 与 MCP 组合攻击面会把"需要本地构造请求"的攻击演化为"诱导用户访问页面即可触发风险"。在已经存在 1-click 静默交易链的前提下，浏览器侧能携带登录态远程触发后端动作，意味着完整资金攻击链可以通过普通网页传播。
