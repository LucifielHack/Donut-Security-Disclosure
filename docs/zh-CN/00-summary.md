## 摘要

这是一份关于 Donut AI 相关产品安全风险的公开披露。披露对象主要包括 Donut Browser 浏览器钱包系统，以及 D0 / OpenClaw AI Agent 系统。

简单说，这些问题说明：当 AI 产品开始接入钱包、交易、自动化执行和用户账户环境时，它已经不再只是一个聊天工具，而是在接近金融执行系统。这样的系统如果缺少严格的权限边界、钱包归属校验和运行环境隔离，就可能把普通用户暴露在资金、隐私和账户控制风险之下。

本次研究共整理 49 个安全问题：Donut Browser 35 个，D0 / OpenClaw 14 个。Donut Browser 部分的核心风险是后端交易构建与执行链路缺少足够的用户隔离；D0 / OpenClaw 部分的核心风险是用户 AI Agent 运行环境、控制面能力和敏感会话材料之间的边界过弱。

在 Donut Browser 测试中，研究验证了跨用户钱包交易请求可以被 Donut 后端继续推进，直到第三方 Turnkey 策略在最后阶段拦截。也就是说，最后阻止第三方资金被转出的关键机制来自外部托管服务，而不是 Donut 自身后端权限模型。

在 D0 / OpenClaw 测试中，研究验证了从前端环境信息、控制面访问到容器内命令执行和敏感数据读取的完整链路。相关验证限定在研究员自有租户 / Pod 范围内，没有对其他用户环境进行横向攻击。

本文面向普通用户、媒体、安全从业者和行业观察者。文中会保留必要的事实、时间线、截图占位、技术结论和公开依据，但不会公开 live token、JWT、Cookie、私钥、反弹 Shell 参数或一键化 exploit 脚本。
