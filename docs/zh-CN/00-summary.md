## 摘要：这些漏洞意味着什么

**对用户的直接威胁：**

> 如果你是 Donut Browser 的用户，任何一个拥有 Donut 账户的人——甚至不需要知道你的密码——只需要知道你的钱包地址，就可以构建一笔从你钱包转出全部资产的交易，并让 Donut 后端自动提交给签名服务。整个过程你不会收到任何确认弹窗、通知或告警。唯一阻止资金被盗的，是第三方托管商 Turnkey 的一条策略规则——不是 Donut 自己写的任何一行代码。

> 如果你是 D0 AI Agent 的用户，你发给 AI 的所有对话、你配置的所有交易权限、你账户的 Session Token，都可以被提取。攻击者可以在你的 Agent 容器里执行任意命令、安装持久化后门、读取你的全部环境信息，并以你的身份调用 Donut 后端 API。

**受影响规模：**
- Donut 钱包系统有 **1,375+ 活跃钱包**，waitlist 用户超过 **16 万人**
- 测试中发现的真实用户钱包持有 **$12,000+** 和 **$76,000+** 资产
- 所有 D0 Agent 用户的容器均存在相同的 RCE 攻击面

**已验证的攻击能力：**
- 已验证：查询任意用户钱包的完整资产、持仓、交易历史
- 已验证：为任意用户钱包构建 SWAP / TRANSFER / DeFi 交易
- 已验证：让后端自动将交易推送到签名服务（无需受害者参与）
- 已验证：11 笔 Solana 主网交易成功上链证明执行链路完整
- 已验证：在 AI Agent 容器内获得 RCE 和交互式反弹 Shell
- 已验证：提取 Session Token、环境变量、内部服务地址
- 已验证：通过 CORS 通配符实现远程一键攻击（受害者点击链接即触发）
