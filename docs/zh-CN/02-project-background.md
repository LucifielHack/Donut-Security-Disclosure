## 项目背景

Donut 是一个 AI + Crypto 产品生态，产品方向包括浏览器、钱包、链上交易辅助、AI Agent 和自动化执行等能力。其核心叙事是让用户通过更自然的方式接入链上资产、策略执行和交易操作。

Donut Browser 承担浏览器与钱包入口角色，涉及用户钱包、资产查询、交易构建、DeFi 操作、限价单和后端交易执行链路。D0 则是 Donut 生态中的 AI Agent 平台，基于 OpenClaw 运行，用户可以在独立环境中配置和运行 Agent。

这类产品的安全风险不同于普通 Web 应用。普通 Web 应用的权限问题可能导致数据泄露或账户功能滥用；而 AI + Crypto 产品一旦连接钱包、签名服务、自动交易和容器运行环境，权限缺陷就可能直接影响资产安全、会话安全和用户运行环境控制权。

因此，对 Donut 这类系统而言，安全边界至少应包括：后端用户身份与钱包归属绑定、交易执行前的强授权确认、最小权限控制、敏感凭据隔离、Agent 运行环境隔离、控制面能力审计，以及异常操作拦截机制。

本次披露讨论的正是这些边界在实际测试中暴露出的系统性问题。
