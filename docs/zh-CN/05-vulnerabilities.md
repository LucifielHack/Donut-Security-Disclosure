## 漏洞索引与明细

本节是公开披露版本的漏洞索引。这里列出漏洞名称、影响面和证据状态，但不放置可直接复用的 exploit、live token、JWT、Cookie、反弹 shell 参数或一键攻击脚本。最终发布前会补充 CVSS/CWE 校对结果和 2026 年 6 月初复测状态。

### Donut Browser 漏洞明细 DB-1 至 DB-35

| 编号 | 漏洞名称 | 严重性 | 类型 | 影响与证据状态 |
|---|---|---|---|---|
| DB-1 | 服务端交易自动签名 / 执行链路缺失强授权边界 | Critical | 资金安全 / 授权缺陷 | 后端可继续推进非当前用户钱包交易请求，链路已通过自有账户链上交易验证，跨用户资金转出由 Turnkey 最后拦截 |
| DB-2 | 跨用户交易构建未绑定当前会话钱包归属 | Critical | IDOR / 业务逻辑 | 交易构建阶段未充分证明 `from_wallet` 属于当前会话用户，可作为资金攻击链入口 |
| DB-3 | 任意钱包资产、持仓、交易历史 IDOR | Critical | 越权访问 / 信息泄露 | 可读取非当前用户钱包的资产组合、成本、持仓和历史信息，已发现高价值钱包样本 |
| DB-4 | CORS 子域通配符与 credentials 组合 | Critical | Web 配置错误 | 恶意子域或被控子域可携带登录态调用敏感 API，放大为远程触发链路 |
| DB-5 | MCP 工具层认证与权限边界不足 | Critical | 认证绕过 / 工具滥用 | MCP 工具可被枚举或直接调用，绕过前端限制并进入工具执行层 |
| DB-6 | Role 参数注入 | High | 输入校验 / 权限提升 | 可伪造 system/assistant 等高权限消息角色，削弱 AI Agent 行为边界 |
| DB-7 | Wallet Service 无认证或弱认证钱包创建 | High | 认证缺失 / 资源滥用 | 可跨 13 条链创建钱包，造成资源消耗、滥用和合规风险 |
| DB-8 | Credits / 使用次数限制绕过 | High | 业务逻辑 | 可绕过 AI Agent 使用额度和订阅限制，降低攻击成本并影响商业模型 |
| DB-9 | AI Agent 配置与 System Prompt 泄露 | High | 信息泄露 / Prompt Injection 原料 | 多个 Agent 的 Prompt、工具配置和调用规则暴露，可用于定向提示词攻击 |
| DB-10 | 跨用户限价单取消 | High | 越权访问 | 订单取消缺少钱包归属校验，可能影响其他用户交易策略和资产操作 |
| DB-11 | DeFi 自动签名交易链路缺少用户确认 | Critical | 资金安全 | DeFi 操作可进入后端签名链路，已通过自有账户交易证明执行路径存在 |
| DB-12 | Admin Plan / 订阅计划信息泄露 | High | 信息泄露 / 业务逻辑 | 管理计划、订阅策略或 taskKey 相关信息暴露，可用于绕过或滥用订阅体系 |
| DB-13 | SQL 查询结构与表结构泄露 | High | 信息泄露 | 错误响应或接口返回暴露 SQL 结构、字段和业务表信息，便于后续攻击建模 |
| DB-14 | 多端点输入验证不足导致 500 错误 | Medium-High | 输入校验 | 对异常类型、对象操作符和边界值处理不足，暴露内部错误处理缺陷 |
| DB-15 | Solana RPC 代理无认证 | Medium | 认证缺失 / 配额滥用 | 可滥用 Donut RPC 资源进行余额、交易和链上数据查询，消耗平台配额 |
| DB-16 | 私有分析数据泄露 | Medium | 信息泄露 | 分析、风险或内部统计数据可被未授权读取，用于用户画像和攻击目标筛选 |
| DB-17 | API 缺失速率限制 | Medium | 配置错误 / 滥用防护不足 | 敏感接口可被批量调用，增加枚举、撞库、资源消耗和链上探测风险 |
| DB-18 | S3 Bucket 或静态资源目录公开 | Medium | 信息泄露 | 公开目录可能暴露报告、构建产物或未来上传的敏感文件 |
| DB-19 | Health / 运维端点信息泄露 | Medium | 信息泄露 | 健康检查端点暴露版本、环境、内存、运行时间、任务队列或集群状态 |
| DB-20 | MCP 工具配置完全公开 | Medium | 信息泄露 | 工具名称、Provider、参数 schema 和能力边界可被枚举，便于构造工具滥用链 |
| DB-21 | OpenAPI / Scalar 文档暴露 | Medium | 信息泄露 | 生产环境暴露完整 API 文档和端点列表，降低攻击枚举成本 |
| DB-22 | 监控报告端点无认证 | Medium | 认证缺失 | 内部监控或报告端点可被未授权访问，泄露运行状态和业务指标 |
| DB-23 | 任意钱包余额探测 | High | 信息泄露 | 可通过错误响应、余额接口或交易构造侧信道探测任意钱包余额 |
| DB-24 | risk-metrics 全平台数据泄露 | Medium | 信息泄露 | 平台级风险指标或统计数据暴露，可用于识别高价值用户和系统状态 |
| DB-25 | beta.donutlabs.dev 第二后端暴露 | High | 攻击面暴露 | 备用或测试后端暴露出独立 API 面，可能绕过主站修复和访问控制 |
| DB-26 | wallet-service 钱包详情无认证查询 | High | 越权访问 | 钱包详情查询缺乏认证或绑定校验，可组合资产枚举与资金攻击链 |
| DB-27 | Admin taskKey / 管理计划路径暴露 | High | 信息泄露 / 权限边界 | 管理类路径或隐藏参数暴露，提供订阅、计划和后台逻辑攻击入口 |
| DB-28 | ORM 操作符注入线索 | Medium | 注入 / 查询污染 | `$ne`、`$gt`、`$regex`、`$in` 等对象操作符触发异常行为，需复测确认影响边界 |
| DB-29 | Helius Webhook 路径仍可访问 | Medium | 认证缺失 / Webhook 暴露 | Webhook 换路径后仍存在可访问入口，增加伪造事件或探测风险 |
| DB-30 | donutlabs.dev 通配符 DNS / 子域解析 | Medium | 配置错误 | 任意子域解析扩大 CORS、钓鱼、恶意页面托管和同站信任边界风险 |
| DB-31 | MCP 工具生态完整枚举 | Medium | 信息泄露 | 可枚举工具生态、调用方式和参数 schema，降低后续攻击链构造成本 |
| DB-32 | HTTP 安全头缺失 | Low | Web 加固不足 | 缺少必要安全响应头，增加点击劫持、内容嗅探和浏览器侧攻击面 |
| DB-33 | Cookie 安全属性不足 | Low | 会话保护不足 | Cookie 缺少 HttpOnly / SameSite 等属性，扩大 XSS 或跨站场景下的会话风险 |
| DB-34 | 前端敏感信息暴露 | Low | 信息泄露 | localStorage、前端包或源码中暴露会话、配置、内部路径或调试信息 |
| DB-35 | 详细错误信息、调试端点和生产指纹暴露 | Low-Medium | 信息泄露 | 错误响应、测试端点和调试信息暴露环境、依赖、内部路径或执行上下文 |

### D0 / OpenClaw 漏洞明细 D0-1 至 D0-14

| 编号 | 漏洞名称 | 严重性 | 类型 | 影响与证据状态 |
|---|---|---|---|---|
| D0-1 | OpenClaw beforeRun / 控制面机制导致 RCE | Critical | 远程代码执行 | 可在自有 D0 Pod 内执行命令，已验证容器内执行与数据提取链路 |
| D0-2 | 环境接口泄露 Gateway 连接信息与 Token | Critical | 凭据泄露 / 权限提升原料 | 前端可获得控制面连接材料，形成从登录态到控制面的直接路径 |
| D0-3 | WebSocket 控制面配置读取 | High | 敏感配置泄露 | 可读取 Agent、heartbeat、beforeRun、注册 Agent 和运行时配置 |
| D0-4 | operator/admin 高权限能力暴露或限制不足 | Critical | 权限边界缺陷 | 认证后可获得过宽控制面能力，扩大配置修改、文件写入和执行风险 |
| D0-5 | Session Token 文件可由容器内进程读取 | High | 会话凭据泄露 | RCE 后可读取会话 token，形成以用户身份访问后端 API 的条件 |
| D0-6 | 容器环境变量泄露 | High | 敏感信息泄露 | 环境变量暴露生产标识、内部服务地址和运行时配置 |
| D0-7 | 内部 ELB / 后端入口暴露 | Medium | 架构泄露 / 纵深攻击面 | 容器配置中出现内部后端入口，为进一步内部探测提供目标 |
| D0-8 | WebSocket Origin / 来源边界不足 | High | WebSocket 认证边界 | 控制面连接对来源和客户端上下文约束不足，增加被跨站或代理滥用风险 |
| D0-9 | Gateway Token 生命周期与轮换不足 | High | 凭据管理缺陷 | Token 静态或轮换不足时，泄露后的控制面访问窗口过长 |
| D0-10 | agents.files.set 等文件写入能力过宽 | Critical | 任意文件/工作区写入 | 可写入 Agent 工作区文件，并与 beforeRun / heartbeat 机制组合为执行链 |
| D0-11 | config.patch 缺乏足够审计和变更保护 | Medium | 配置完整性 / 审计缺失 | 关键运行配置可被修改，缺少明确审计、回滚和完整性保护证据 |
| D0-12 | Pod 间网络隔离仍需进一步授权验证 | High | 隔离边界风险 | 当前已验证自有 Pod 内 RCE；横向移动边界需进一步授权测试确认 |
| D0-13 | heartbeat 脚本完整性校验不足 | High | 执行完整性缺陷 | heartbeat 前置脚本可成为持久化执行点，绕过 AI 模型层安全审查 |
| D0-14 | JWT 有效期与敏感凭据暴露风险 | Medium | 会话安全 | token 生命周期较长且可被容器内读取，增加失窃后的滥用窗口 |
